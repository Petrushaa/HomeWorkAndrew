# План рефакторинга архитектуры (Clean Architecture)

В этом плане описан процесс приведения текущей структуры проекта к стандартам слоистой архитектуры, как у преподавателя.

## Шаг 1: Выделение слоя Adapters (Внешние интеграции)
Внешние сервисы (MinIO/S3) не должны лежать в бизнес-логике (`services/`).
1. Создать папку `adapters/storage/`.
2. Переместить файл `services/storage.py` в `adapters/storage/s3.py`.
3. Обновить импорты во всех файлах, где использовался `StorageService`.

## Шаг 2: Разделение слоя API (Роутеры)
Файл `main.py` не должен содержать реализации эндпоинтов.
1. Создать файл `api/system.py` и перенести туда эндпоинты `/health` и `/info`.
2. Создать файл `api/tasks.py` и перенести туда все эндпоинты, начинающиеся с `/v1/tasks/` (включая работу с комментариями и MinIO).
3. Создать файл `api/router.py`, в котором будут собраны все роутеры проекта (`auth_router`, `system_router`, `tasks_router`) в один `api_router`.

## Шаг 3: Очистка main.py
Файл приложения должен быть тонким и отвечать только за инициализацию.
1. Убрать все `app.get`, `app.post` и т.д. из `main.py`.
2. Подключить единый `api_router` из `api/router.py`.
3. Оставить инициализацию `FastAPI()`, настройки CORS, `lifespan` и обработчики исключений.

## Завершение
Запустить тесты (pytest), чтобы убедиться, что перемещение файлов не сломало импорты и логику.
